System Pipeline

Five stages, one operator-facing story.

The architecture operates as a single, cohesive decision-support chain. Click the stages below to see how data flows through the system.

Main operational data flow Simulation-backed advisory flow Operator feedback loop
Detection

SCADA anomaly extraction

Raw rows are scanned for alarm codes, stop flags, and abnormal operating signals.

Output: anomaly rows and grouped events
Aggregation

Daily health metadata

Events are converted into interpretable turbine-day indicators such as duration, severity, and health label.

Output: a clean metadata table for engineering interpretation
Explanation

LLM maintenance report

The language model turns metadata into a concise maintenance summary with advice for the operator.

Output: natural-language diagnosis support
Gate

RL advisory control filter

Health status determines whether the optimization layer is blocked, constrained, or allowed as advisory only.

Output: a safe optimization posture
Action

Unified operator briefing

The final page combines alarms, explanation, RL posture, and the next human action.

Output: one actionable briefing instead of fragmented tools
