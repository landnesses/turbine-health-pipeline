Example Turbine-Day Briefings

Three cases showing health-aware operator guidance.

The goal is not only anomaly detection. The briefing should clarify what operators should do next under each health condition.

Turbine A

ATTENTION state

Constrained

Abnormal event lasted several hours, with alarm code detected.

Health: ATTENTION Long abnormal duration

Recommended action

Review SCADA logs and confirm turbine operational status.

Control policy

RL is allowed only as limited advisory support.

Turbine B

ALARM state

Disabled

Stopping event triggered with pitch lubrication alarm.

Health: ALARM Stop event present

Recommended action

Inspect pitch lubrication system and prioritize maintenance.

Control policy

RL optimization is disabled in alarm state.

Turbine C

Repeated ALARM pattern

Disabled

Repeated pitch system alarms indicate persistent maintenance priority.

Health: ALARM Repeated alarm family

Recommended action

Launch maintenance investigation before optimization discussion.

Control policy

Optimization advice is suppressed until condition recovery.

TakeawayThe same architecture produces different operator actions for different health states.
