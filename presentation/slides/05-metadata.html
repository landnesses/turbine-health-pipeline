Health Metadata Layer

Explainability begins with interpretable turbine-day features.

Instead of sending opaque internal representations downstream, the system aggregates event evidence into a metadata layer that engineers can read, validate, and audit.

Interpretable turbine-day features form a white-box schema — each field maps to measurable evidence.

Representative metadata fields
FieldMeaningWhy it matters
event_countNumber of abnormal events in a turbine-dayMeasures frequency of operational disruption
abnormal_duration_minTotal minutes under abnormal stateSeparates transient issues from persistent conditions
dominant_alarmMost important alarm family observedAnchors the maintenance explanation in evidence
severity_levelAlarm / Attention / NormalDirectly gates the advisory optimisation posture
health_labelDaily health classificationProvides the operator-facing summary state
Why this supports explainable AI

Each feature maps to a measurable turbine property or event statistic.

The LLM summary can cite these fields instead of inventing opaque reasoning paths.

The RL layer is constrained by a health state that is inspectable by engineers.

White-box framing: explainability is achieved by explicit data design before model output is generated.
TakeawayMetadata is the bridge between raw telemetry and trustworthy operator guidance.
